# ToDo Bee — Панель управления личной эффективностью (Фронтенд-клиент)

Данный репозиторий содержит клиентскую часть (фронтенд) веб-приложения для тайм-менеджмента и трекинга продуктивности **«ToDo Bee»**, разработанного в рамках курсовой работы по дисциплине «Веб-программирование». 

Приложение спроектировано как высокопроизводительный, легковесный SPA-подобный интерфейс в оригинальной визуальной «пчелиной» стилистике.

---

## Особенности архитектуры фронтенда

*   **Чистый технологический стек (Vanilla CSS3 & JS)**: Отказ от использования тяжелых клиентских фреймворков (таких как React, Angular или Vue) гарантирует максимальную производительность, минимальный размер бандла и мгновенную отрисовку интерфейса.
*   **Модульный JS (ES Modules)**: Весь клиентский код разделен на строго разграниченные изолированные модули, поддерживающие нативный импорт в современных браузерах.
*   **Сверхбыстрая сборка**: Использование инструмента нового поколения **Vite** обеспечивает мгновенный запуск сервера разработки и эффективную оптимизацию статических ресурсов при финальной сборке.
*   **Гибкое позиционирование (CSS Grid & Flexbox)**: Адаптивная разметка, разработанная с нуля без фреймворков верстки. Приложение оптимизировано под мобильные устройства (от 320px), планшеты и десктопные мониторы. Сайдбар навигации на смартфонах трансформируется в эргономичную нижнюю панель (bottom-navigation).
*   **Микроинтерактивы на CSS3**: Применение плавных CSS-переходов (`transition`) для визуального отклика интерфейса (эффект плавного подъема карточек `translateY` при наведении, анимации раскрытия модальных окон, плавное зачеркивание чекбоксов).
*   **Централизованное управление темой**: Цветовая палитра вынесена в CSS Custom Properties (CSS-переменные), что обеспечивает простую поддержку смены тем оформления и глобального брендинга.

## Функциональные разделы интерфейса

Клиентское приложение содержит следующие специализированные разделы (страницы):

1.  **Календарный планировщик задач (`index.html` / `planner.html`)**:
    *   Интерактивная сетка дней текущей недели для оперативного распределения нагрузки.
    *   Создание задач с указанием заголовка, подробного описания, даты дедлайна и приоритета (низкий, средний, высокий).
    *   Декомпозиция крупных целей на вложенные чек-листы (подзадачи) со свободным изменением статусов.
2.  **Дневник рефлексии и эмоционального здоровья (`diary.html`)**:
    *   Текстовое поле для ведения ежедневных записей ментального состояния.
    *   Шкала эмоциональной рефлексии (Mood Tracker) с оценкой настроения от 1 до 5.
    *   Привязка записей к календарным датам с возможностью редактирования прошлых дней.
3.  **Аналитическая статистика (`stats.html`)**:
    *   Сводные числовые показатели (процент завершенных задач, стрик непрерывной активности).
    *   Визуальные интерактивные графики и диаграммы за последние 7 дней для анализа взаимосвязи настроения и продуктивности.
4.  **Личный кабинет и профиль (`profile.html`)**:
    *   Редактирование личных данных пользователя (отображаемое имя).
    *   Инструмент интерактивной загрузки аватарного изображения с последующим сохранением на сервере или в облачном хранилище.
    *   Функция каскадного удаления учетной записи и всех связанных с ней данных.

## Установка и локальный запуск

Для работы с проектом требуется предустановленная программная платформа [Node.js](https://nodejs.org/) версии 18 или выше.

### 1. Перейдите в каталог проекта
```bash
cd bzz_do/bzz_do
```

### 2. Установите зависимости
Установите Vite и вспомогательные пакеты сборщика:
```bash
npm install
```

### 3. Запустите сервер разработки
Запустите локальный веб-сервер Vite с поддержкой горячей перезагрузки стилей и логики:
```bash
npm run dev
```
После успешного запуска перейдите в веб-браузере по адресу, указанному в терминале (обычно [http://localhost:5173/](http://localhost:5173/)).

### 4. Сборка для публикации (Production)
Скомпилируйте и оптимизируйте статические файлы для развертывания на хостинге:
```bash
npm run build
```
Собранные и минифицированные файлы будут сохранены в каталоге `/dist`.
